# QNN Conversion Reference

## Scope
Use when `{FLOW}=QNN`.

> 🚦 **Path selector**
>
> | Situation | Pipeline | Script |
> |-----------|----------|--------|
> | Default / all new work | **Flow A** — ONNX → DLC → .bin | `run_pipeline.py` |
> | DLL pipeline / needs `.dll` | **Flow C (legacy)** — ONNX → C++/BIN → DLL → .bin | `run_pipeline_legacy.py` |
>
> Default = Flow A: `qairt-converter` → `qairt-quantizer` (opt) → `qai_dev_gen_contextbin.py --model <file>.dlc`. No VS ARM64 env.
> Below = **legacy Flow C**. DLC pipeline → SKILL.md Step 4 / `run_pipeline.py --help`.

> **Wrappers** (`qai_convert_fp.py` / `qai_convert_int.py`): Flow C only. Auto-detect QAIRT paths; pass `--preserve_io`. Flow A → `run_pipeline.py`.

## ✅ Verified Flow (Legacy Flow C — QAIRT 2.45, WoS ARM64, FP16)

> Flow C. Flow A → `run_pipeline.py --help` / SKILL.md Step 4.

Verified: Snapdragon X Elite (V73), QAIRT 2.47.1.260610.

> ⚠️ `.cpp` patching may be needed — see `operator_patching.md`.
> ⚠️ Disable training-only branches before ONNX export (e.g. `aux_logits`). See `model_export_validation.md`.

### Prerequisites

> ⚠️ All paths from `${APP_ROOT}\data\config\qairt_env.json` (auto-generated by `Setup.bat`). Steps 1–2: `python_x64_venv` (x86_64 3.10); Step 4+: `python_arm64_venv` (ARM64 3.13). Never hardcode.

```bat
@echo off
for /f "delims=" %%S in ('powershell -NoProfile -Command "(Get-Content ${APP_ROOT}\data\config\qairt_env.json | ConvertFrom-Json).qairt_sdk_root"') do set QNN_SDK_ROOT=%%S
for /f "delims=" %%P in ('powershell -NoProfile -Command "(Get-Content ${APP_ROOT}\data\config\qairt_env.json | ConvertFrom-Json).python_x64_venv"') do set PYTHON_X64=%%P\Scripts\python.exe
for /f "delims=" %%V in ('powershell -NoProfile -Command "(Get-Content ${APP_ROOT}\data\config\qairt_env.json | ConvertFrom-Json).vs_vcvarsall"') do set VS_VCVARSALL=%%V
for /f "delims=" %%T in ('powershell -NoProfile -Command "(Get-Content ${APP_ROOT}\data\config\qairt_env.json | ConvertFrom-Json).vc_targets_path"') do set VCTargetsPath=%%T

set PYTHONPATH=%QNN_SDK_ROOT%\lib\python;%PYTHONPATH%
set PATH=%QNN_SDK_ROOT%\lib\aarch64-windows-msvc;%QNN_SDK_ROOT%\bin\aarch64-windows-msvc;%QNN_SDK_ROOT%\bin\x86_64-windows-msvc;%PATH%

call "%VS_VCVARSALL%" arm64
```

### Step 1: ONNX → C++/bin

```bat
REM --output_path stem = graph name in Step 3
%PYTHON_X64% "%QNN_SDK_ROOT%\bin\x86_64-windows-msvc\qnn-onnx-converter" ^
    --input_network ${WORKSPACE}\<model>\<model>.onnx ^
    --no_simplification ^
    --output_path ${WORKSPACE}\<model>\output\<model>.cpp ^
    --float_bitwidth 16 ^
    --preserve_io
```

**Expected:**
```
INFO - Model CPP saved at: output/<model>.cpp
INFO - Model BIN saved at: output/<model>.bin
INFO - Conversion complete!
```

> ⚠️ `--float_bw 16` deprecated → use `--float_bitwidth 16`. `--no_simplification` recommended for WoS. Opset 18 supported. Large models: pass `.onnx`, converter finds `.onnx.data` automatically.

### Step 2: C++/bin → ARM64 DLL

```bat
REM aarch64-windows-msvc (NOT x86_64) — requires vcvarsall arm64
%PYTHON_X64% "%QNN_SDK_ROOT%\bin\aarch64-windows-msvc\qnn-model-lib-generator" ^
    -c ${WORKSPACE}\<model>\output\<model>.cpp ^
    -b ${WORKSPACE}\<model>\output\<model>.bin ^
    -o ${WORKSPACE}\<model>\output ^
    -t windows-aarch64
```

**Expected:**
```
INFO - Library name         : <model>
INFO - Generated model library path: output\ARM64\<model>.dll
```

### Step 3: DLL → Context Binary

```bat
copy "%QNN_SDK_ROOT%\lib\aarch64-windows-msvc\QnnHtp.dll"            ${WORKSPACE}\<model>\output\
copy "%QNN_SDK_ROOT%\lib\hexagon-v73\unsigned\libqnnhtpv73.cat"       ${WORKSPACE}\<model>\output\
copy "%QNN_SDK_ROOT%\lib\hexagon-v73\unsigned\libQnnHtpV73Skel.so"    ${WORKSPACE}\<model>\output\

REM Generator resolves .cat/.so relative to CWD; script sets cwd=output_dir.
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_dev_gen_contextbin.py ^
    --model ${WORKSPACE}\<model>\output\ARM64\<model>.dll ^
    --output_dir ${WORKSPACE}\<model>\output ^
    --binary_file <model>_fp16 ^
    --auto-config
```

**Expected:**
```
Starting stage: Graph Preparation Initializing
...
Starting stage: Completion
Completed stage: Completion (515 us)
```
→ `${WORKSPACE}\<model>\output\<model>_fp16.bin`

> ⚠️ `--auto-config` generates backend JSON configs. `graph_names` = stem of `--output_path` from Step 1. Use `--config_file` for custom.

### Configuration Files (manual, without --auto-config)

Templates → [`context_binary.md`](context_binary.md). `graph_names` MUST = stem of `--output_path` from Step 1; mismatch → `unable to find graphName`.

### Common Errors Quick Reference

| Error | Cause | Fix |
|-------|-------|-----|
| `unable to find graphName` | `graph_names` ≠ DLL graph name | `--auto-config` or set `graph_names` = `--output_path` stem |
| `MODEL_INVALID_ARGUMENT_ERROR` | Same | Same |
| `Graph Compose failure` | Config mismatch / unsupported op | Check graph_names + operator support |
| `No CMAKE_C_COMPILER` | VS ARM64 env missing | `vcvarsall.bat arm64` in same `.bat` |
| `Unable to load backend: QnnHtp.dll` | DLL not in CWD/PATH | Copy `QnnHtp.dll` to working dir |
| `Backend version mismatch` | Wrong SDK QnnHtp.dll | Same SDK version for all steps |
| `Wrong number of Parameters 5` | VS ARM64 env missing | `vcvarsall.bat arm64`; `.cat`/`.so` in CWD. Else → `operator_patching.md` |
| `VCTargetsPath.vcxproj` / `BaseOutputPath not set` | VCTargetsPath → BuildTools | Read `vc_targets_path` from `qairt_env.json`; `where.exe MSBuild.exe` |
| `CMake Error: Failed to run MSBuild` | BuildTools can't compile ARM64 | VS 2022 **Community**; set VCTargetsPath |

## Float conversion (FP16/FP32)

```bat
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_fp.py ^
  --onnx ${WORKSPACE}\<model>\<model>.onnx ^
  --precision 16 ^
  --output-root ${WORKSPACE}\<model>\output ^
  --no-simplification
```

`--no-simplification` maps to `--no_simplification`; recommended for WoS/QAIRT 2.45.

### Explicit input dimensions

For dynamic shapes, use `--input-dim`:

```bat
REM Wrapper (all comma-separated)
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_fp.py ^
  --onnx ${WORKSPACE}\<model>\<model>.onnx ^
  --precision 16 ^
  --output-root ${WORKSPACE}\<model>\output ^
  --input-dim input,1,3,64,64

REM Direct converter (SPACE between name and dims)
%PYTHON_X64% "%QNN_SDK_ROOT%\bin\x86_64-windows-msvc\qnn-onnx-converter" ^
  --input_network model.onnx ^
  --input_dim "input 1,3,512,512" ^
  --output_path output\<model>.cpp ...
```

> ⚠️ Direct: `--input_dim "name dim1,dim2"` (space separates name). Wrapper: `--input-dim name,dim1,dim2` (all commas).

Multiple inputs — repeat flag:
```bat
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_fp.py ^
  --onnx model.onnx ^
  --input-dim input1,1,3,224,224 ^
  --input-dim input2,1,10
```

## Quantized conversion — Legacy Flow C (`qai_convert_int.py`)

> ⚠️ Flow C only. Flow A quantized → [`model_quantization.md`](model_quantization.md).

```bat
REM A16W8 (recommended for vision)
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_int.py ^
  --input_network ${WORKSPACE}\<model>\<model>.onnx ^
  --input_list ${WORKSPACE}\<model>\calib\calibration_list.txt ^
  --output-root ${WORKSPACE}\<model>\output ^
  --act_bw 16 --weight_bw 8

REM A8W8
%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_int.py ^
  --input_network ${WORKSPACE}\<model>\<model>.onnx ^
  --input_list ${WORKSPACE}\<model>\calib\calibration_list.txt ^

%PYTHON_X64% ${APP_ROOT}\factory\chat_features\model-builder\scripts\qai_convert_int.py ^

  --input_list ${WORKSPACE}\<model>\calib\calibration_list.txt ^

```

> ⚠️ `input:=` is Flow C only. Flow A uses plain paths. → [`model_quantization.md § Step 2`](model_quantization.md).

### Default: Flow A — `run_pipeline.py` (ONNX → DLC → .bin)

Reads `qairt_env.json`, runs `qairt-converter` → `qairt-quantizer` (opt) → `qai_dev_gen_contextbin.py --model <file>.dlc`. **No VS ARM64 env.**

```bat
REM FP16 (default)
<python_x64_venv>\Scripts\python.exe ${APP_ROOT}\factory\chat_features\model-builder\scripts\run_pipeline.py ^
  --model ${WORKSPACE}\<model>\<model>.onnx ^
  --output ${WORKSPACE}\<model>\output --precision fp16

REM bf16 (Transformer/LLM)
<python_x64_venv>\Scripts\python.exe ${APP_ROOT}\factory\chat_features\model-builder\scripts\run_pipeline.py ^
  --model ${WORKSPACE}\<model>\<model>.onnx ^
  --output ${WORKSPACE}\<model>\output --precision bf16

REM W8A8 + CLE + per-channel + encoding dump
<python_x64_venv>\Scripts\python.exe ${APP_ROOT}\factory\chat_features\model-builder\scripts\run_pipeline.py ^
  --model ${WORKSPACE}\<model>\<model>.onnx ^
  --output ${WORKSPACE}\<model>\output --precision w8a8 ^
  --calib_list ${WORKSPACE}\<model>\calib\calibration_list.txt ^
  --cle --per_channel --dump_encoding
```

### Legacy: Flow C — `run_pipeline_legacy.py`

Same CLI. Internally: `qnn-onnx-converter` + `qnn-model-lib-generator` (VS ARM64) + `qai_dev_gen_contextbin.py --model <file>.dll`. Debug → `qai_convert_fp.py` / `qai_convert_int.py`.

### `run_pipeline.py` — Full Argument Reference

**Legacy-compatible flags:**

| Argument | Req | Default | Description |
|----------|-----|---------|-------------|
| `--model` | Y | — | Input ONNX path |
| `--output` | N | `qairt_output` | Output directory |
| `--precision` | N | `fp16` | `fp32`\|`fp16`\|`bf16`\|`w4a8`\|`w4a16`\|`w8a8`\|`w8a8b8`\|`w8a16`\|`w16a16`. Exclusive with `--act_bw`/`--weight_bw`. |
| `--act_bw` | N* | — | Activation bits (4/8/16). Requires `--weight_bw`. |
| `--weight_bw` | N* | — | Weight bits (4/8/16). Requires `--act_bw`. |
| `--bias_bw` | N | — | Bias bits (8/32). Unset ⇒ FP32. |
| `--calib_list` | Y (quant) | — | Calibration file. **Plain paths, no `input:=`.** |
| `--input_dim` | N | — | e.g. `"input 1,3,512,512"` → `qairt-converter --source_model_input_shape`. |
| `--config` | N | — | HTP `backend_extensions.json` path. |
| `--htp_version` | N | `v73` | `v73` (X Elite) or `v81` (X2 Elite). |
| `--skip_contextbin` | N | False | Output DLC only. |
| `--no_simplification` | N | False | → `qairt-converter --onnx_skip_simplification`. |

**DLC-path flags** (not in legacy):

| Argument | Default | Description |
|----------|---------|-------------|
| `--io_config <yaml>` | — | Per-tensor layout/dtype/color-encoding |
| `--quant_overrides <json>` | — | AIMET encoding overrides / mixed precision |
| `--strip_quant` | False | Stripped FP DLC (`DLC_STRIP_QUANT`) |
| `--per_channel` | False | Per-channel weight quant (Conv). Rec for W8A8 |
| `--per_row` | False | Per-row weight quant (Matmul/FC) |
| `--calib_method` | `min-max` | `{min-max, sqnr, entropy, mse, percentile}` |
| `--percentile_value` | 99.99 | For `--calib_method percentile` |
| `--cle` | False | Cross-Layer Equalization. Rec for W8A8 |
| `--dump_encoding` | False | Per-tensor quant encodings JSON |
| `--generic_graph` | (no-op) | Deprecated → `--soc_optimized` |
| `--soc_optimized` | False | SoC-specific HTP. OFF = cross-platform |
| `--profiling` | False | HTP optrace profiling |

*`--act_bw`/`--weight_bw` must pair.

**Output:** `{stem}_{precision}.bin` (e.g. `inception_v3_fp16.bin`)

**Precision → bitwidth:**

| `--precision` | act_bw | weight_bw | bias_bw |
|---------------|--------|-----------|--------|
| `fp16` | — | — | — |
| `fp32` | — | — | — |
| `w8a16` | 16 | 8 | — (FP32) |
| `w8a8` | 8 | 8 | — (FP32) |
| `w8a8b8` | 8 | 8 | 8 |
| `w4a16` | 16 | 4 | — (FP32) |
| `w4a8` | 8 | 4 | — (FP32) |
